WebShield — Privacy Policy

Effective date: October 28, 2025

WebShield (the "Extension") is developed and maintained by Emir Abdullah Alakus (github.com/Abdullah-js). This Privacy Policy explains what information the Extension collects (if any), how it is used, and how you can contact the developer.

Summary (short)

WebShield does not collect or transmit personally identifiable information by default. Settings and preferences are stored locally in your browser using Chrome's storage API. Any optional feature that requires remote resources or data transfer will be explicit in the UI and require your consent prior to usage.

Data collection

No automatic personal data collection: By default, the Extension does not collect, upload, or store personal data to any remote server. The Extension runs locally in your browser and uses local storage only for configuration and preferences.

Local storage and what is stored

The Extension stores non-sensitive configuration values and small preferences (for example: toggles you enable, domain trust lists, UI preferences). These are stored using Chrome's local storage (`chrome.storage.local`) and are kept on the user's device.

Optional remote features (explicit opt-in)

If you choose to enable optional features that require remote access (for example, pattern updates, optional cloud-sync, or anonymous analytics), the Extension will:

  • Ask for your explicit consent before enabling the feature;
  • Describe exactly what data is sent and why;
  • Allow you to disable the feature and delete any remote data that was created under that feature.

Third-party links and services

WebShield may include links to external sites (for example, Ko-fi). Clicking an external link will open that site in a new tab. We do not control third-party privacy practices; please review their privacy policies separately.

Cookies and similar technologies

The Extension does not set or rely on browser cookies for user tracking. Any website you visit may set cookies independent of this Extension.

Security

We follow best-effort security practices: code runs within the extension context, remote requests (if any) use HTTPS, and sensitive patterns/data should be kept server-side if you enable server sync. However, no system is perfectly secure — treat the Extension and any optional remote features with appropriate caution.

Retention

Local settings are retained until you uninstall the Extension or clear your browser data. Any optional remote data (if you opt-in) is retained according to the description you are shown when you enable that feature; you may request deletion where applicable.

Children

The Extension is not intended for children under 13. We do not knowingly collect personal information from children under 13.
